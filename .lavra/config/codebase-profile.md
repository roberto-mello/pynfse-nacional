## Architecture & Structure

**Directory Layout:**
```
src/pynfse_nacional/
  __init__.py       # public API surface (re-exports via __all__)
  client.py         # NFSeClient — orchestration facade
  models.py         # Pydantic domain models (DPS, NFSe, Prestador, Tomador, ...)
  xml_builder.py    # XMLBuilder — DPS/event XML construction
  xml_signer.py     # XMLSignerService — xmlsec signing, cert handling
  pdf_generator.py  # standalone PDF renderer (optional extra)
  constants.py      # enums (Ambiente), API URL maps
  exceptions.py     # NFSeAPIError, NFSeCertificateError
  utils.py          # gzip/base64 compression, CNPJ/CPF formatters
```

**Design Patterns:**
- Facade: `NFSeClient` is single entry point
- Pipeline: DPS model -> XMLBuilder -> XMLSignerService -> httpx -> NFSeResponse
- Builder: `XMLBuilder` uses private `_add_*` methods per DPS section
- Context manager: `_get_client()` wraps httpx.Client mTLS lifecycle
- Optional extras: pdf_generator isolated behind install extra

**Dependency layering (no cycles):**
exceptions < utils/constants < models < builders/signers < client

**Key observations:**
- xml_builder.py uses stdlib xml.etree.ElementTree; xml_signer.py uses lxml.etree — potential namespace divergence risk
- XMLSignerService.compress_encode duplicates logic in utils.py (minor SRP drift)

## Conventions & Testing

**Code Style:**
- PEP 8, line length 88, Ruff rules: E, F, I, N, W
- snake_case functions/variables, CamelCase classes
- Domain terms in Portuguese (Prestador, Tomador, Servico, DPS, NFSe)

**Test Framework:**
- pytest with asyncio_mode = "auto"
- unittest.mock.patch for HTTP mocking
- Integration tests skipped without NFSE_TEST_CERT_PATH / NFSE_TEST_CERT_PASSWORD env vars

**Naming Patterns:**
- Validators: `validate_<field_name>` (e.g. validate_cnpj, validate_cep)
- Private helpers: `_validate_*` prefix
